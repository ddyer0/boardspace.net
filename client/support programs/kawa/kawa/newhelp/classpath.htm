Kawa automatically sets the basic classpath for all the JDK classes and Kawa classes automatically. You don't need to set any classpath under your system for Kawa to function properly.

However, it is necessary to set classpath under Kawa if you are programming with Swing under JDK 1.1 or if you are using some non-JDK packages like database drivers as part of your project. In these situations please set the project classpath to include any directories or zip or jar files.

Project Classpath -

Select Project/Classpath to display the classpath dialog shown above. Select New to add a new classpath value. A new value can be either a directory or a file(zip or jar) or it can be one of the standard Kawa parameters. Please select Add button to add the new classpath value to the classpaths listbox. Kawa also allows you to temporarily disable or enable a classpath from the project by checking or unchecking the checkbox next to the value. Finally, the order of the classpath is as shown in the classpaths listbox. Please use move up or move down to rearrange the listbox.

Project classpath is always added ahead of the rest of the classpaths.
