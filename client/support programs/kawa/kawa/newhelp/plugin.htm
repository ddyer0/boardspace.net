Kawa plugin is the new plugin architecture which lets you write Java programs to control Kawa. It is like a Java API to extend Kawa functionality.

Kawa plugin architecture is based on Java's JNI interface. As part of this architecture Kawa exposes 5 Java objects - KawaApp, KawaProject, KawaEditor, KawaFile and KawaFolder.

Some of the above objects defines event objects to indicate certain events. Please refer to KawaAppEvent, KawaProjectEvent and KawaFileEvent for more details on Kawa plugin events.

Each of these objects define certain methods to externally control and drive Kawa. This is a first iteration of this architecture and we hope to grow this architecture more and more based on your feedback.

The design goals for the first release of this architecture was better source control integration and ability to setup custom builds for some large projects. We think we have achived both these goals with the first iteration.

The only environment where Kawa plugin can run is when launched by Kawa. You can launch a plugin command in 2 ways - setup a custom plugin command or use single run plugin menu option.

Kawa plugins are enabled automatically. Please check Customize/Options/Editor to disable kawa plugins.

All plugin classes are packed into kawaclasses.zip in Kawa folder and Kawa automatically sets the classpath to this file. All plugin documentation files and plugin sample files are copied under /plugin folder in the Kawa installation directory.

We hope to come up with more sample plugin programs very soon. We also hope that you will get as much excited as we are about this new architecture. Please provide as much feedback as you can on this new feature.
