Please install Kawa as a single user setup on the server first and follow these steps on each client workstation that needs to access Kawa from the server.

Please make a directory on the client workstation for all Kawa configuration files.

Start Kawa on the client and goto Customize/Options/Directories and set the Kawa configuration directory to the directory you just created on the client.

Following are the files in the user directory -

    kawa.pkg - Package tab data file

    user.ipg - User help index file

    user.cmd - User custom commands file

    kawa.hpg - Help tab file

    kawa.proj - Kawa project file

If any of these files are not present on the server there is no need to copy the same on the client. Also, it is important to delete these files from the server once copied to the client.

Following files should go to Windows/System directory.

It is important to verify the version information of each DLL before copying it to System directory.

    mfc42.dll

    msvcrt.dll

    msvcrt40.dll

    oleaut32.dll

    olepro32.dll

    stdole2.tlb

All the remaining files can stay in the server directory. Also, when you create a shortcut on the client to run Kawa, please make sure the current directory is the directory on the server where Kawa files are residing. This is important.
