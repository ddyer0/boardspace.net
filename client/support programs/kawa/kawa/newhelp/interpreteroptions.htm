Project interpreter options is now divided into 2 tabs - basic and advanced. Please refer to the screen shot below for each tab options. Please refer to JDK documentation on java for more information on each of these options. Following is a brief description of some important options.

  • Java command line options is for passing command line arguments to the Java program.
  • Java Class name to run is the name of the class to run. Please refer to How to Run an applet or application for more details on how the class name for running an application is decided.
  • Execute program directory option when enabled will indicate the working directory for starting the Java application or applet.
  • Custom options are sent as is to the interpreter including the debugger. Please specify the new -X properties for JDK 1.2 as custom option.

Advanced Options

  • Remote debugger host name and password option can be used to start a remote debug session. When this field is checked Kawa will try and connect to remote debugger with the remote host and password as typed in the field. Please note the format of this field. It should be typed "-host -password " Kawa will ignore any interpreter options run classname or the main class selection when remote debugger field is checked. All the current project breakpoints are set with the remote debugger and if any breakpoint is hit Kawa will open the source file for the breakpoint.
  • Property values are sent to the interpreter with a -D option. These properties are also sent to the debugger

Please refer to JDK documentation on javac tools for all the other interpreter switches. Project interpreter options will supercede any global interpreter options set by selecting Customize/Project Root options menu item. Global interpreter options are a subset of the project interpreter options and is applied universally to all the projects.
