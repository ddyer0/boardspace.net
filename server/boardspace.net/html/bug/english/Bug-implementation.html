How to Identify "bugs" 

Bug identifies bugs as the same shape, regardless of orientation or mirroring.  According to the internet, the
number of such bugs for each size is as follows:
11
21
33
47
522
682
7333
81448

And sizes even larger than 8 are theoretically possible.  Generating (or recognising equivalent shapes is a tricky problem.

Here's how Boarspace does it.  

represent a "bug" as a set of cells.

for each cell in the set, calculate the distance to each of the other cells in the set, using a modified version
of the board drawing algorithm.  It's vital that connections for A to B give exactly the same distance whenever
A and B are in the same relation to one another counting hexes between them.

for each cell, characterize the pattern of the 6 adjacent cells.  There are 10 such patterns.

for each distance/from/to triplet, construct an integer.  Sort them to give a canonical order.

construct a zorbrist hash with a table of random numbers and the sorted connection numbers.

the intent is that the hash uniquely identifies the bug, whatever its orientation, mirror image,  or position.

verify these proceedures by counting that the correct number of  bugs are eventually encountered, no less
and certainly no more.
