BoardSpace utilise Java !

This browser does not have a Java Plug-in.

Si l'applet Java ne se lance pas, c'est un problème à régler en priorité.  Nos jeux nécessitent un explorateur adapté à Java. La plupart des explorateurs récents depuis Netscape 3.0 ou Explorer 3.0 ... et leurs successeurs supportent Java, mais leur installation n'inclut pas toujours automatiquement Java. 

Les principales causes de "pas de java" dans un explorateur sont :
  • Java n'est pas installé, vous devez le télécharger. Ou Java est installé, mais dans une très ancienne version. Boarspace est programmé de façon à fonctionner avec toutes les versions hormis les plus anciennes - une version 1.1 ou plus devrait suffire.  Visitez Java.com pour tester l'installation de Java sur votre machine et au besoin réaliser cette installation.
  • Dans tous les explorateurs, il est possible de désactiver Java à partir des préférences et parfois l'option par défaut est de le désactiver, toutes les installations n'incluent pas Java par défaut. Si votre explorateur est assez récent pour supporter Java (sinon, avec quoi surfez-vous donc ?), vérifiez que java est activé dans les préférences de votre explorateur. Pour des raisons connues seulement de développeurs, ceci semble être différent pour chaque version de chaque explorateur.  Java.com présente les solutions pour rendre java activé sur la plupart des explorateurs.
  • Certaines versions d'explorateurs pour AOL ne supportent pas Java. Utilisez un explorateur directement, sans passer par AOL.
  • Sur Macintosh, Internet Explorer est fourni sans l'installation de Java (au moment où j'écris ces lignes), mais vous pouvez télécharger Java chez Apple and l'installer.
Deux sites pour vous aider à tester votre installation de la plateforme Java sur votre machine : Java.com  et  Fitwise.com

Firewalls, Routeurs, Ports et Proxies, Pertes de connexion

Si vous obtenez un message du type "connexion impossible" dans la zone de chat du lobby :  l'applet Java doit pouvoir ouvrir un conexion tcp sortante sur le port 2255.  La plupart des firewalls et routeurs sont configurés pour accepter cela ; mais certaines entreprises ou écoles et quelques firewalls personnels (blackice, zone alarm) peuvent bloquer ce port.  Si la fenêtre de l'applet est ouverte, mais que la connexion est impossible, voyez le problème avec le port 2255.  L'applet peut également échouer à se connecter si elle a été chargée à partir d'ailleurs que Boardspace.net, ce qui peut être un effet de bord de la configuration de certains proxies. 

Si vous obtenez le message rcv 221 errConnection error alors vous êtes correctement connecté au serveur, mais la connexion a été rejetée.  La cause la plus plausible est que vous vous êtes accidentellement connecté au serveur, sans passer par la procédure normale de connexion.  Ceci peut se produire si vous utilisez l'icône "Page précédente" de votre explorateur ou si votre explorateur a mis la page de connexion en cache (et donc ne discute pas avec le seveur pour se connecter).  Vérifiez les réglage du cache de votre explorateur.

Si vous perdez la connexion en cours de partie : pas de panique, vous pouvez vous reconnecter et oursuivre la partie. Ceci peut se produire pour tout un tas de raisons, aussi imprévisibles que surprenantes. Observez le petit logo  à côté de votre pendule (et celle de votre adversaire).  La petite aiguille doit chager de position en moins de dix secondes et si ce n'est pas le cas, la couleurs passe du vert vers le rouge. Après deux minutes d'inactivité, un "timeout" est déclaré et vous pouvez vous reconnecter.

Lenteur de connexion : Une fois l'applet téléchargée, la largeur de bande nécessaire au lobby et aux jeux est très modeste, et n'importe quelle connexion internet doit faire l'affaire. Un problème déjà rencontré est que des firewalls tels que Zone Alarm interfèrent avec le débit de données vers du serveur sans le bloquer totalement. Nous ignorons pourquoi, mais désactiver Zone Alarm résout le problème. Pour tester cette situation, changez le type d'une salle de jeu, disons de "Salle de jeu" à "Salle d'entraînement".  Le changement doit s'effectuer très rapidement, dans la seconde. Si ce n'est pas le cas, vérifiez ce type de problème.

Si vous pouvez jouer normalement vos parties, mais que le calcul du score final échoue : pour clore une partie normalement, l'applet doit pouvoir lancer des requêtes HTTP sur les ports 80 ou 4321.  Le port 80 est normalement utilisé par lex explorateurs, il est donc usuellement ouvert, mais certains sites forcent l'utilisation d'un proxy et ferment le port 80.  Occasionnellement, il y aura aussi échec en cas de grose affluence sur le site, ou en raison d'évènements imprévisibles sur l'internet.  si le port 80 est bloqué defaçon permanente, tentez d'accéder à Boardspace par : http://www.boarspace.net:4321/


Explorateurs, Plateformes and Versions Java

L'applet Java qui fait tourner les jeux fonctionne avec n'importe quel explorateur, mais aucun explorateur aucune plateforme ne naît égale.  Aucun n'est parfait, mais certains ont davantage tendance aux problèmes que d'autres.  

Windows (toute version)

Internet Explorer est de loin le choix le plus courant, mais aussi le plus sensible aux problèmes, particulièrement les erreurs de type "out of memory".
Netscape (anciennes versions) ne fait pas mieux.
Mozilla or Netscape (versions récentes) sont meilleurs.  Ces versions utilisent un plugin Java très fiable.

D'autres choix conviennet aussi , comme  HotJava et Opera, mais leur emploi est limité et insuffisant pour parler de leur fiabilité. 

Macintosh et Linux

Les quatre principaux explorateurs (IE, Netscape, Mozilla and Safari) utilisent le pluging Java, donc s'il fonctionne, c'est de façon similaire. Le mot clé étant si.  Les developpeurs d'OS et les développeurs d'explorateurs ne jouent pas toujours dans la même cour, aussi peut-il y avoir des problèmes quand certaines anciennes versions n'en supportent pas certaines autres.   Assez peu de joueurs utilisent des mac ou des plateformes linux, tout ce que l'on peut dire c'est que les configurations classiques sont en général ok.

Anciennes versions Java
Boardspace fonctionne avec  Java VM version 1.4.2 ou supérieure.

Sur Macintosh, certaines anciennes versions de MRJ sont buggées et ne fonctionnent pas du tout, même si elles peuvent fonctionner avec certains autres sites.
